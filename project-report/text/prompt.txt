============================================================

你是一名资深软件架构师、UML 建模专家和游戏引擎架构分析师。请根据以下项目事实，为 Java Swing 2D 游戏项目《暮光之旅》生成一张清晰、专业、适合课程设计/软件设计文档使用的“总体架构设计图 / 系统结构图”。

请优先输出 Mermaid flowchart 代码，方向使用 TB 或 LR，图中必须体现分层架构、核心模块职责、主要调用/依赖关系、运行时数据流和持久化数据流。请不要虚构不存在的模块，不要把所有类平铺成一团，要按层次分组。节点文字使用中文为主，可在括号中保留关键类名。

项目技术栈：
- Java 桌面应用，Gradle 构建，主类为 org.example.App。
- GUI 使用 Java Swing/AWT。
- 数据持久化使用 SQLite，仓库类为 MapRepository，默认数据库路径为 ~/.hnsfgame/maps.db。
- JSON 导入导出和对象扩展字段使用 org.json。
- 日志使用 logback，测试使用 JUnit 5。

项目核心定位：
- 这是一个 2D 游戏引擎原型，包含可运行游戏窗口、关卡/地图编辑器、对象系统、物理碰撞、动态光照、输入控制、状态机、地图持久化和程序化关卡生成。

请按以下层次绘制：

1. 启动与应用层
- App：应用入口，判断命令行参数。
- 游戏模式：App.startGame() 创建 MapRepository、LevelManager、GameWorld、SwingGamePanel、JFrame。
- 编辑器模式：App.startEditor() 创建 MapRepository、LevelManager、GameWorld，并打开 EditorWindow。
- GameRuntimeActions：状态机和 UI 菜单调用的运行时动作接口，用于加载关卡、下一关、打开编辑器、退出、生成程序化关卡。

2. 表现与主循环层
- SwingGamePanel：游戏主面板，持有 GameWorld、GameInputController、Swing Timer、Camera、SettingsRepository、ControlHintsOverlay、MinimapOverlay、GoalOverlay、DebugManager、AITestManager。
- 帧循环 onFrame：读取输入 -> 状态机处理输入 -> AI 测试更新 -> GameWorld.update -> Camera 跟随玩家 -> repaint。
- paintComponent：缩放逻辑视口 -> GameWorld.render -> 控制提示/调试/小地图/目标叠加层渲染。
- Camera：负责大地图视口跟随玩家。
- LightingManager：动态光照和探索雾渲染。
- SoundManager：播放伤害、射击、菜单、治疗、胜利等音效。

3. 核心游戏世界层
- GameWorld：运行时核心聚合对象，管理世界尺寸、背景、重力、胜利条件、击杀/收集进度、重生点、屏幕震动。
- GameWorld 持有 EntityManager、PhysicsEngine、LightingManager、SoundManager、Camera、GameStateMachine。
- EntityManager：管理所有 GameObject 生命周期、按 GameObjectType 缓存分组、分层渲染世界对象和 UI 对象。
- PhysicsEngine：AABB 碰撞、移动解析、实体与固体场景碰撞检测。
- LevelManager：注册内置关卡 tutorial、demo-map、level-1 到 level-4、procedural-forest、procedural-cave、air-raid-demo；创建内置关卡和程序化关卡。
- ProceduralLevelGenerator + NoiseGenerator：生成森林/洞穴地图数据。

4. 输入与状态机层
- GameInputController：统一处理键盘、鼠标和语义动作，调用状态机或直接控制玩家移动、射击、跳跃、冲刺、菜单导航、体素建造/销毁。
- KeyboardManager / MouseManager：记录原始输入状态。
- InputActionMapper：将按键/鼠标映射到 InputAction。
- InputAction：MOVE、SHOOT、PAUSE、JUMP、SPRINT、CYCLE_COLOR、CYCLE_PROJECTILE_TYPE、MENU_CONFIRM 等语义动作。
- GameStateMachine 接口与 DefaultGameStateMachine 实现：管理 MENU、PLAYING、DIALOG、PAUSED、GAMEOVER、SETTLEMENT 状态；根据状态处理输入、创建暂停菜单、失败菜单、胜利菜单，检查胜利/失败。
- GameStateContext：把 GameWorld、GameInputController、GameSettings、GameRuntimeActions、deltaSeconds 组合成状态机输入上下文。
- GameSettings：运行时设置接口，由 SwingGamePanel 实现，提供 FPS、UI 字号、重力、光照、音量等配置。

5. 游戏对象模型层
- GameObject 接口：定义 name、type、position、size、color、active、update、updateInactive、render。
- BaseObject：实现 GameObject 的通用字段。
- ActorObject：角色基类，提供生命值、最大生命值、攻击力、速度、受伤、治疗、死亡动画、血条/体力条渲染。
- PlayerObject：玩家，支持移动加速度、冲刺、跳跃、射击、弹种切换、颜色切换、经验等级、光球视野增强、补色伤害机制。
- MonsterObject：怪物，支持追击、飞行、远程攻击、投弹、掉落治疗、复活、不同 MonsterKind。
- ProjectileObject：投射物，支持 ProjectileType，如普通弹、照明弹、炸弹、穿透弹、追踪弹、冰霜弹、激光弹、寻敌弹。
- SceneObject：场景块，支持 solid/background、可破坏、耐久、失去支撑倒塌、踩踏损坏。
- WallObject、BoundaryObject、VoxelObject：场景类对象的特殊实现。
- ItemObject：可拾取物品，支持 health、speed、lightorb、coin/gem/xp、shield。
- GoalObject：终点/传送门。
- MenuObject、DialogObject：UI 层对象。
- GameObjectFactory：GameObject 与 ObjectData 之间的创建/序列化工厂。

6. 持久化与数据层
- MapData：地图 DTO，包含名称、尺寸、背景颜色/预设/模式/图片、重力、胜利条件、目标击杀/收集数量、ObjectData 列表。
- ObjectData：对象 DTO，包含 type、name、x/y、width/height、color、solid、background、texturePath、material、extraJson。
- MapDataMapper：GameWorld <-> MapData <-> JSONObject 的转换；导入/导出 JSON；Base64 编码背景图；加载时用 GameObjectFactory 还原对象。
- MapRepository：SQLite 仓库，负责 maps 和 map_objects 表的初始化、迁移、保存、加载、列表查询。
- SettingsRepository：保存运行时设置，如 FPS、分辨率、UI 字号、重力、光照、音量、键位绑定。

7. 地图编辑器层
- EditorWindow：Swing 地图编辑器主窗口，包含对象库、预览区、地图属性、对象属性、工具面板。
- EditorGamePanel：编辑器预览面板。
- MapEditorController：处理选择、建造、破坏、拖拽、复制、删除、撤销/重做、网格吸附。
- EditorOverlay：显示网格、选中框、编辑模式等叠加信息。
- EditorBounds：约束对象坐标和尺寸在地图范围内。
- LevelAutoScanner：扫描关卡配置问题，如缺少玩家/终点、击杀/收集目标不合理等。

必须画出的关键关系：
- App -> SwingGamePanel -> GameWorld。
- App -> MapRepository、LevelManager、MapDataMapper。
- SwingGamePanel -> GameInputController、Camera、SettingsRepository、Overlays、DebugManager、AITestManager。
- SwingGamePanel 每帧调用 GameInputController.processInputs()、GameWorld.update()、GameWorld.render()。
- GameWorld 聚合 EntityManager、PhysicsEngine、LightingManager、SoundManager、GameStateMachine。
- GameInputController 依赖 KeyboardManager、MouseManager、InputActionMapper，并通过 GameStateContext 调用 DefaultGameStateMachine。
- DefaultGameStateMachine 读取 GameWorld 对象和碰撞结果，驱动菜单/对话/暂停/胜利/失败流程，并通过 GameRuntimeActions 请求 App 加载关卡、打开编辑器、生成程序化地图。
- EntityManager 管理 GameObject 集合；GameObject 由 BaseObject、ActorObject 及各具体对象实现。
- MapDataMapper 连接运行时模型 GameWorld/GameObject 与 DTO MapData/ObjectData；GameObjectFactory 连接 ObjectData 和具体 GameObject。
- MapRepository 将 MapData 保存到 SQLite 的 maps、map_objects 表；也从 SQLite 读取 MapData。
- EditorWindow 使用 GameWorld 预览和修改地图，通过 MapDataMapper + MapRepository 保存/加载。
- LevelManager 创建内置关卡，ProceduralLevelGenerator 创建程序化关卡，输出 MapData。

- GameWorld.updateInactiveObjects(deltaSeconds) 处理失活对象后台逻辑，如光球复生。
- Camera.update(world, player) 跟随玩家。
- inputController.finishFrame() 清除瞬时输入。
- repaint()。

3. 渲染流程：
- SwingGamePanel.paintComponent() 设置逻辑视口缩放。
- GameWorld.render(Graphics2D)。
- GameWorld 渲染背景 -> 应用 Camera 偏移和屏幕震动 -> EntityManager.renderWorld() -> LightingManager.render() -> EntityManager.renderUI()。
- SwingGamePanel 继续渲染 ControlHintsOverlay、DebugManager、MinimapOverlay、GoalOverlay。

4. 地图数据流：
- LevelManager / ProceduralLevelGenerator 生成 MapData。
- MapRepository 从 SQLite 保存/读取 MapData。
- MapDataMapper 在 GameWorld、MapData、JSONObject 之间转换。
- GameObjectFactory 在 GameObject 和 ObjectData 之间转换。
- EditorWindow 通过 MapDataMapper + MapRepository 保存、加载、导入、导出地图。

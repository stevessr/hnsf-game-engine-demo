# 系统架构说明

## 1. 架构总览

项目可以分为以下几个运行层：

```text
App
 ├─ 游戏模式：SwingGamePanel → GameWorld → EntityManager / PhysicsEngine / LightingManager
 ├─ 编辑器模式：EditorWindow → MapEditorController → GameWorld
 └─ 数据层：MapRepository / MapDataMapper / SettingsRepository
```

核心目标是把“**游戏运行**”和“**关卡编辑**”放在同一个代码库中，但通过不同窗口和控制器分离。

## 2. 启动流程

### 2.1 `org.example.App`

`App.main()` 会先检查命令行参数：

- 如果第一个参数是 `editor`，进入编辑器模式
- 否则进入游戏模式

### 2.2 游戏模式

游戏模式的启动顺序大致如下：

1. 创建 `MapRepository`
2. 创建 `LevelManager`
3. 准备内置关卡
4. 用关卡数据装载 `GameWorld`
5. 创建 `SwingGamePanel`
6. 装配 `JFrame`、菜单栏和运行时动作
7. 启动面板定时器，进入帧循环

### 2.3 编辑器模式

编辑器模式的启动顺序：

1. 创建编辑器窗口
2. 创建编辑器预览面板
3. 绑定 `MapEditorController`
4. 初始化属性面板、对象选择和保存逻辑
5. 通过鼠标/键盘事件进行对象放置、拖拽、删除和缩放

## 3. 运行时主循环

`SwingGamePanel` 是整个游戏的帧驱动入口：

1. 轮询键盘和鼠标输入
2. 让状态机处理当前输入
3. 更新世界中的对象
4. 进行碰撞和物理解析
5. 渲染世界、光照和 UI

这个过程把“输入 → 状态判断 → 世界更新 → 渲染输出”串成了一个完整循环。

## 4. 主要数据流

### 4.1 地图加载

地图加载通常遵循：

`MapRepository` / 内置关卡生成 → `MapData` → `MapDataMapper.applyToWorld()` → `GameWorld`

### 4.2 地图保存

地图保存通常遵循：

`GameWorld` → `MapDataMapper.fromWorld()` → `MapRepository.saveMap()` / JSON 导出

### 4.3 设置保存

`SwingGamePanel` 在运行中会把：

- 分辨率
- 帧率
- UI 字体大小
- 重力开关
- 光照开关
- 音量
- 按键绑定

持久化到 `SettingsRepository`。

## 5. 渲染管线

当前渲染可理解为三层：

1. **世界层**
   - 背景、地形、玩家、怪物、投射物、道具等

2. **光照层**
   - 由 `LightingManager` 叠加环境光和局部光源

3. **UI 层**
   - 菜单、对话框、提示信息、迷你地图、目标提示

这种分层可以避免 UI 被光照影响，同时便于将来加入更多覆盖层。

## 6. 状态机

`DefaultGameStateMachine` 管理的主要状态包括：

- `MENU`
- `PLAYING`
- `DIALOG`
- `PAUSED`
- `GAMEOVER`
- `SETTLEMENT`

状态机不仅决定“当前能做什么”，也决定“哪些菜单对象应该显示、隐藏或居中”。

## 7. 设计特点

- **耦合可控**：世界对象不直接操作窗口，窗口也不直接操作对象细节。
- **扩展性较好**：对象创建、地图导入导出、编辑器属性编辑都通过统一映射层处理。
- **利于测试**：输入、物理、数据映射、状态机都能独立测试。

# 初级软件实作报告（项目分析版）

## 1、实作项目简介

`primary_software` 是一个基于 Java Swing 的 2D 游戏原型项目，入口类为 `org.example.App`。

项目支持两种启动方式：

- **游戏模式**：启动 `SwingGamePanel`，进入主游戏循环。
- **编辑器模式**：通过命令行参数 `editor` 打开 `EditorWindow`，进入地图编辑器。

从源码结构看，项目已经形成了较完整的模块划分：

- 游戏世界与关卡管理：`lib.game`
- 输入系统：`lib.input`
- 对象模型：`lib.object`
- 物理系统：`lib.physics`
- 渲染系统：`lib.render`
- 状态机：`lib.state`
- 地图编辑器：`lib.editor`
- 持久化：`lib.persistence`
- 通用管理器：`lib.manager`

整体风格是“**单窗口桌面游戏 + 内置编辑器 + SQLite 地图持久化**”。

如果你要继续深入某个子模块，可以直接查看：

- [`modules/README.md`](modules/README.md)
- [`reference/README.md`](reference/README.md)

## 3、项目设计

### 3.1 总体架构设计

当前工程可以抽象为以下几层：

1. **入口层**
   - `org.example.App`
   - 负责区分游戏模式与编辑器模式，并组装主窗口。

2. **运行时层**
   - `lib.game.GameWorld`
   - `lib.render.SwingGamePanel`
   - `lib.state.DefaultGameStateMachine`
   - 负责驱动更新、渲染、状态切换和菜单行为。

3. **对象模型层**
   - `lib.object.*`
   - 负责表示玩家、怪物、投射物、场景块、菜单、对话框、触发器等。

4. **输入与交互层**
   - `lib.input.*`
   - 负责键盘/鼠标状态采集，以及动作映射。

5. **物理与碰撞层**
   - `lib.physics.*`
   - 负责 AABB 碰撞、移动解析和边界限制。

6. **渲染层**
   - `lib.render.*`
   - 负责摄像机、背景、光照、迷你地图和 UI 覆盖层。

7. **持久化层**
   - `lib.persistence.*`
   - 负责地图保存、加载、JSON 导入导出和程序化关卡生成。

8. **编辑器层**
   - `lib.editor.*`
   - 负责地图编辑、对象属性编辑、预览与撤销/重做。

### 3.2 类设计

#### 3.2.1 `lib.game`

- `GameWorld`：世界状态容器，管理对象、背景、光照、重生点、胜利条件和屏幕震动。
- `LevelManager`：内置关卡和程序化关卡的组织者。
- `WinConditionType`：胜利条件枚举。

#### 3.2.2 `lib.state`

- `GameState`：菜单、游戏、对话、暂停、失败、结算等状态枚举。
- `GameStateContext`：把世界、输入、设置和运行时动作聚合成状态上下文。
- `DefaultGameStateMachine`：负责状态切换和菜单/对话/暂停逻辑。
- `GameRuntimeActions`：运行时动作集合。
- `GameSettings`：供面板和状态机共享的设置接口。

#### 3.2.3 `lib.object`

- `GameObject`：对象统一接口。
- `BaseObject`、`ActorObject`：对象基础抽象。
- `PlayerObject`、`MonsterObject`、`ProjectileObject`、`ItemObject`、`GoalObject` 等：具体游戏实体。
- `SceneObject`、`WallObject`、`BoundaryObject`、`VoxelObject`：环境与地形对象。
- `MenuObject`、`DialogObject`：UI 对象。
- `GameObjectFactory`：对象与 DTO 的双向转换核心。

#### 3.2.4 `lib.render`

- `SwingGamePanel`：主渲染面板与帧循环。
- `Camera`：摄像机视口。
- `LightingManager`：动态光照管理。
- `BackgroundAssets`、`SpriteAssets`、`TextureManager`：素材和贴图管理。
- `ControlHintsOverlay`、`GoalOverlay`、`MinimapOverlay`：覆盖层 UI。

#### 3.2.5 `lib.persistence`

- `MapRepository`：SQLite 持久化仓库。
- `MapDataMapper`：`GameWorld` / `MapData` / `JSONObject` 之间的转换工具。
- `SettingsRepository`：运行参数持久化。
- `ProceduralLevelGenerator`、`NoiseGenerator`：程序化关卡生成。

#### 3.2.6 `lib.editor`

- `EditorWindow`：编辑器主窗口。
- `MapEditorController`：编辑行为控制器。
- `EditorGamePanel`：编辑器预览渲染面板。
- `EditorOverlay`：编辑器辅助图层。
- `LevelAutoScanner`：关卡自动扫描。

## 4、系统实现

### 4.1 界面

项目的 UI 主要由 `SwingGamePanel` 和 `EditorWindow` 两条线组成：

- `SwingGamePanel`
  - 负责主游戏窗口渲染
  - 管理帧率、输入监听、设置持久化
  - 将世界渲染、光照和 UI 覆盖层组合在一起

- `EditorWindow`
  - 负责地图编辑器完整界面
  - 提供对象类型、地图属性、背景、碰撞、怪物、触发器等编辑控件
  - 支持预览、保存、加载和对象选择/拖拽/缩放

### 4.2 控制

控制逻辑由三部分组成：

1. **输入采集**
   - `KeyboardManager`
   - `MouseManager`
   - `GameInputController`

2. **动作映射**
   - `InputActionMapper`
   - 将具体按键和鼠标按钮映射成语义动作，例如移动、跳跃、射击、暂停、切换弹种等。

3. **状态控制**
   - `DefaultGameStateMachine`
   - 控制菜单、对话、游戏、暂停、失败和结算之间的流转。

其中 `App` 中的菜单逻辑表明，程序支持：

- 重启关卡
- 导入地图
- 导出当前地图
- 打开地图编辑器
- 关于说明

### 4.3 资源管理

资源管理分成三类：

- **对象资源**：`GameObjectFactory` 将对象数据转回运行时对象。
- **地图资源**：`MapRepository` 与 `MapDataMapper` 负责地图的落盘与恢复。
- **运行时资源**：`SoundManager`、`TextureManager`、`SpriteAssets`、`BackgroundAssets` 等负责音效与贴图。

## 6、总结

### 6.1 设计亮点

1. **模块分层清晰**
   - 入口、世界、输入、物理、渲染、持久化、编辑器职责分开。

2. **状态驱动明确**
   - 游戏状态机将菜单、对话、暂停和结算行为统一管理。

3. **编辑器和游戏联动**
   - 关卡不仅能玩，还能直接在编辑器中编辑并保存。

4. **数据转换完整**
   - `MapDataMapper` 连接运行时世界、持久化模型和 JSON 文件。

### 6.2 不足与可扩展方向

1. 当前主要面向桌面端 Swing，跨平台和现代化 UI 扩展空间有限。
2. 编辑器控件较多，后续可以继续抽象成更统一的属性面板。
3. 程序化生成、光照、声音和战斗系统还有进一步调优空间。
4. 可补充更多自动化测试，尤其是 UI 和集成场景。

# 核心模块说明

## 1. 游戏世界 `lib.game`

### `GameWorld`

`GameWorld` 是运行时核心容器，负责：

- 保存世界尺寸、背景和当前关卡设置
- 持有 `EntityManager`
- 持有 `PhysicsEngine`
- 持有 `LightingManager`
- 管理重生点、目标计数、击杀数和收集数
- 维护胜利条件和失败原因
- 提供屏幕震动等战斗反馈

### `LevelManager`

`LevelManager` 负责关卡组织与加载：

- 内置关卡：`tutorial`、`demo-map`、`level-1` ~ `level-4`
- 程序化关卡：`procedural-forest`、`procedural-cave`
- 演示关卡：`air-raid-demo`、`showcase-demo`

它通过 `MapDataMapper` 把关卡数据变成 `GameWorld` 可直接使用的运行时对象。

### `WinConditionType`

胜利条件枚举，支持不同类型的通关逻辑，例如：

- 到达目标点
- 击杀所有怪物
- 达成指定击杀数
- 收集指定数量物品
- 清空所有物品

## 2. 实体管理 `lib.manager`

### `EntityManager`

`EntityManager` 负责所有对象的生命周期和分层渲染：

- 统一保存对象列表
- 按 `GameObjectType` 做缓存
- 支持增删清空
- 支持按类型查询
- 支持世界层和 UI 层分开渲染

它是整个项目里非常关键的性能优化点。

### 其他管理器

- `SoundManager`：音效播放与音量控制
- `DebugManager`：调试信息与可视化辅助
- `AITestManager`：AI 测试或自动化辅助

## 3. 输入系统 `lib.input`

### `InputActionMapper`

它把“具体按键/鼠标按钮”映射为“语义动作”：

- 移动：`MOVE_UP` / `MOVE_DOWN` / `MOVE_LEFT` / `MOVE_RIGHT`
- 战斗：`JUMP` / `SHOOT` / `CYCLE_PROJECTILE_TYPE`
- UI：`MENU_PREVIOUS` / `MENU_NEXT` / `MENU_CONFIRM` / `PAUSE`
- 编辑器：`VOXEL_BUILD` / `VOXEL_DESTROY`

### `KeyboardManager` / `MouseManager`

负责记录：

- 当前是否按下
- 本帧刚刚按下
- 本帧刚刚释放

这些状态会被状态机和运行时动作直接使用。

### `GameInputController`

统一协调键盘和鼠标输入，并向上层暴露当前动作状态。

## 4. 物理系统 `lib.physics`

### `PhysicsEngine`

`PhysicsEngine` 负责：

- 目标位置的合法性校验
- 与障碍物的碰撞检测
- 世界边界限制
- 按 X / Y 轴分步解析移动

### `Aabb`

用于表示轴对齐包围盒，是碰撞检测的基础数据结构。

### `MovementResult`

用于返回物理解析后的结果，包括：

- 最终坐标
- X / Y 是否被阻挡
- 阻挡对象

## 5. 对象模型 `lib.object`

### 基础结构

- `GameObject`：统一接口
- `BaseObject`：基础实现
- `ActorObject`：可移动/可交互对象基类

### 典型对象

- `PlayerObject`：玩家
- `MonsterObject`：怪物
- `ProjectileObject`：投射物
- `ItemObject`：道具
- `GoalObject`：终点
- `WallObject` / `BoundaryObject`：墙体和边界
- `SceneObject`：通用场景块
- `MenuObject` / `DialogObject`：UI 对象

### 工厂与 DTO

- `GameObjectFactory`：对象与 DTO 之间的双向转换
- `GameObjectType`：对象类型枚举
- `MonsterKind`、`ProjectileType`、`TriggerAction`：细分类枚举
- `dto/MapData`、`dto/ObjectData`：持久化和导入导出的中间模型

## 6. 渲染系统 `lib.render`

### `SwingGamePanel`

主渲染和主循环面板，负责：

- 输入监听
- 定时刷新
- 场景绘制
- 光照叠加
- UI 叠加
- 设置保存

### `Camera`

控制视口位置与显示范围。

### 覆盖层

- `ControlHintsOverlay`
- `GoalOverlay`
- `MinimapOverlay`
- `DebugManager`

这些类让游戏运行信息可以叠加到主画面之上。

## 7. 状态系统 `lib.state`

### `DefaultGameStateMachine`

这是状态逻辑的核心：

- 控制状态合法转换
- 控制菜单显示/隐藏
- 控制暂停与恢复
- 控制对话框和结算界面

### `GameStateContext`

把世界、输入、设置和运行时动作组合起来，供状态机处理输入时使用。

## 8. 关键协作关系

一个比较典型的协作链路是：

`InputActionMapper` → `GameInputController` → `DefaultGameStateMachine` → `GameWorld` → `EntityManager` / `PhysicsEngine` → `SwingGamePanel`

这条链路说明项目并不是“面向单个类”的写法，而是围绕系统职责进行拆分的。

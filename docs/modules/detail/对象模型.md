# 对象模型详细分析

## 1. 模块职责

对象模型是整个项目的业务核心。

它负责描述：

- 玩家
- 怪物
- 投射物
- 物品
- 终点
- 墙体与边界
- 场景块
- 菜单
- 对话框
- 触发器
- 生成器
- 体素

## 2. 基础层次

### 2.1 `GameObject`

统一接口，包含：

- 名称
- 类型
- 位置
- 大小
- 颜色
- 是否激活
- `update()`
- `updateInactive()`
- `render()`

### 2.2 `BaseObject`

基础实现，提供：

- 通用位置和尺寸管理
- 颜色与激活状态
- 材质、贴图路径
- 垂直速度等基础字段

### 2.3 `ActorObject`

适合“会战斗的实体”，它额外管理：

- 生命值
- 最大生命值
- 攻击力
- 速度
- 死亡动画
- 受伤与治疗

## 3. 玩家对象 `PlayerObject`

玩家对象是功能最丰富的实体之一，支持：

- 等级与经验
- 冲刺与体力
- 加速度与减速度
- 油门力度
- 弹种切换
- 跳跃
- 射击
- 炸弹蓄力
- 颜色切换
- 光球 / 视野增强效果
- 受伤无敌时间
- 确认方向的最后朝向

玩家对象不仅处理移动，还处理战斗和反馈效果。

## 4. 怪物对象 `MonsterObject`

怪物对象不仅有血量和攻击力，还带有较丰富的 AI 行为特性：

- 怪物种类 `MonsterKind`
- 是否敌对
- 是否远程攻击
- 是否飞行
- 是否可重生
- 是否投弹
- 重力适应百分比
- 掉落治疗道具
- 跳跃、闪避、巡逻、追击、撤退、射击等行为

`MonsterObject` 本身已经接近“小型 AI 单元”。

## 5. 场景对象 `SceneObject`

场景对象兼具“静态关卡块”和“动态结构块”两种身份。

它支持：

- 固体/非固体
- 背景/前景
- 可破坏
- 耐久度
- 失去支撑后坍塌
- 根据步数破碎

还支持多种特殊外观：

- 虚空
- 云
- 树
- 标牌

这使得场景对象不只是地图装饰，而是可以影响战斗和移动的实体。

## 6. UI 对象

### 6.1 `MenuObject`

菜单对象管理：

- 标题和副标题
- 可选项列表
- 选中项
- 悬停项
- 列数
- 滚动与可见行数

它可以用于主菜单、暂停菜单、选项菜单、胜利菜单等。

### 6.2 `DialogObject`

对话框对象管理：

- 说话者名字
- 消息文本
- 字体大小

它支持自动换行渲染，适合教程、剧情和提示信息。

## 7. 交互对象

### 7.1 `ItemObject`

物品对象支持：

- 类型名称 `kind`
- 数值 `value`
- 提示消息 `message`
- 是否可再生
- 重新出现延迟

根据测试和工厂逻辑，物品可以表现为：

- 回复生命
- 增加视野
- 收集类奖励

### 7.2 `TriggerObject`

触发器对象支持：

- 目标名
- 触发动作 `TriggerAction`
- 是否仅触发一次

常见用途包括：

- 激活/关闭目标对象
- 设置复活点
- 显示/隐藏关卡目标
- 重生玩家

### 7.3 `SpawnerObject`

生成器对象负责周期性生成怪物，配置包括：

- 怪物类型
- 生成间隔
- 最大存活数
- 每波生成数
- 生成半径
- 偏移量
- 组 ID

它适合构建动态敌群。

## 8. 投射物对象 `ProjectileObject`

投射物是战斗系统里变化最多的对象之一。

它支持：

- 速度
- 伤害
- 重力影响
- 爆炸
- 爆炸半径
- 引信时间
- 生命时长
- 光照范围
- 追踪
- 穿透
- 是否揭示探索迷雾

这对应了项目里的多种弹种，例如：

- 普通弹
- 照明弹
- 爆破弹
- 穿透弹
- 追踪弹
- 冰霜弹
- 激光弹
- 追猎弹

## 9. 枚举与类型识别

对象类型相关枚举主要包括：

- `GameObjectType`
- `MonsterKind`
- `ProjectileType`
- `TriggerAction`

它们是对象工厂、编辑器下拉框和序列化的基础。

## 10. 工厂与序列化

### 10.1 `GameObjectFactory.toObjectData()`

把运行时对象转成 `ObjectData`。

### 10.2 `GameObjectFactory.fromObjectData()`

把 `ObjectData` 还原为运行时对象。

### 10.3 `extraJson`

这是对象扩展字段的关键。

它允许把不同对象的专属属性放进统一的 JSON 里，例如：

- 玩家等级、体力、弹种
- 怪物种类、是否飞行、掉落治疗
- 菜单选项和字体
- 触发器动作
- 投射物速度、穿透、追踪

## 11. 相关测试

对象模型测试覆盖面非常广，说明这个模块是项目最核心的部分之一。

重点包括：

- 玩家升级
- 怪物死亡
- 场景对象规范化
- 玩家移动与渲染
- 腿部渲染不越界
- 治疗特效
- 虚空伤害
- 蝙蝠、蜘蛛、幽灵等特殊渲染
- 道具拾取与重生
- 可复活怪物
- 怪物巡逻、跳跃、躲避、追击、撤退
- 飞行怪物与轰炸怪物行为
- 背景与场景渲染缓存
- 投射物与可破坏场景交互
- 多种弹种的行为与序列化

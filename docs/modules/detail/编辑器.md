# 编辑器详细分析

## 1. 模块职责

编辑器是一个独立但和游戏世界共享模型的工具。

它的目标是让用户可以：

- 可视化放置对象
- 修改对象属性
- 调整地图大小
- 切换背景与胜利条件
- 保存 / 导入 / 导出地图

主要类：

- `EditorWindow`
- `EditorGamePanel`
- `EditorOverlay`
- `MapEditorController`
- `EditorBounds`
- `LevelAutoScanner`

## 2. `EditorWindow`

`EditorWindow` 是完整的 Swing 编辑器窗口。

它把界面分成几个部分：

- 左侧：选择与编辑区
- 中间：预览区
- 右侧：地图 / 对象 / 工具标签页

### 2.1 地图属性

地图属性可编辑内容包括：

- 地图名称
- 宽高
- 重力开关
- 重力强度
- 胜利条件
- 目标击杀数
- 目标收集数
- 背景预设
- 背景模式
- 背景颜色
- 背景图片

### 2.2 对象属性

对象属性面板覆盖了大量对象类型特有配置，比如：

- 玩家
- 怪物
- 物品
- 触发器
- 生成器
- UI 对象
- 场景对象

这意味着编辑器不是“只会放方块”，而是已经接近通用地图制作器。

## 3. `MapEditorController`

这是编辑器逻辑的核心控制器。

### 3.1 编辑模式

它支持三种模式：

- `SELECT`：选择
- `BUILD`：建造
- `ERASE`：删除

### 3.2 交互特性

控制器支持：

- 鼠标悬停提示
- 选中对象拖拽
- 对象缩放
- 网格吸附
- 方向键微调
- 复制对象
- 删除对象
- 撤销 / 重做

### 3.3 命令栈

编辑器采用命令栈结构管理撤销重做：

- `AddCommand`
- `RemoveCommand`
- `MoveCommand`
- `ResizeCommand`
- `PropertyCommand`

这是典型的命令模式，适合编辑器类场景。

## 4. 编辑器覆盖层

### `EditorOverlay`

负责显示：

- 网格
- 选中对象轮廓
- 操作模式
- 提示信息

这层让编辑器更容易理解当前状态。

## 5. 选择边界与布局

### `EditorBounds`

它负责把编辑对象的边界限制在世界内。

从测试命名看，它至少包含：

- 位置归一化
- 超出世界时缩小或裁剪

这对拖拽和缩放尤其重要。

## 6. 关卡自动检查

### `LevelAutoScanner`

这个类更像是编辑辅助校验工具，用于检查关卡是否缺少关键元素，例如：

- 玩家
- 终点
- 目标数量是否合理

它可以帮助发现“地图可以保存，但不能正常通关”的问题。

## 7. 编辑器与游戏共用模型

编辑器和游戏共享同一套对象模型，因此：

- 编辑器里改的对象可以直接在游戏里运行
- 游戏世界也可以直接快照成地图数据

这是整个项目的一大优点。

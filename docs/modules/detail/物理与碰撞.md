# 物理与碰撞详细分析

## 1. 模块职责

这个模块负责最基础的“空间合法性”计算：

- 角色能否移动到目标位置
- 会不会撞墙
- 会不会越界
- 当前坐标是否与别的实体重叠

主要类：

- `Aabb`
- `MovementResult`
- `PhysicsEngine`

## 2. `Aabb`

`Aabb` 是轴对齐包围盒。

它是项目里碰撞系统的基本几何单元，通常用于：

- 与场景块碰撞
- 与玩家或怪物碰撞
- 判定子弹命中

## 3. `PhysicsEngine.resolveMovement()`

这是最重要的方法之一。

### 3.1 输入

它接收：

- 移动对象
- 目标坐标
- 世界宽高
- 障碍物列表

### 3.2 处理流程

1. 将目标坐标限制在世界边界内
2. 用“移动范围包围盒”筛出可能碰撞的障碍物
3. 先沿 X 轴一步一步推进
4. 再沿 Y 轴一步一步推进
5. 返回最终可达位置和阻挡信息

这种逐步推进的方式可以减少“穿墙”问题。

### 3.3 返回结果

`MovementResult` 会记录：

- 最终 X / Y
- X / Y 是否被阻挡
- 阻挡对象

这样上层逻辑可以基于碰撞结果做后续处理，例如：

- 受到伤害
- 触发反弹
- 停止移动

## 4. `collidesAt()`

这个方法用于快速判断对象在某个位置是否会和任一障碍物重叠。

常见用途：

- 攻击判定
- 放置检查
- 地图生成时的合法性检查

## 5. `findCollisions()`

这个方法直接返回当前与给定对象相交的所有活跃对象。

它适合：

- 玩家与敌人接触
- 弹体与目标接触
- 触发器激活
- 道具拾取

## 6. 与 `GameWorld` 的协作

`GameWorld` 不是自己做复杂碰撞，而是把工作交给 `PhysicsEngine`：

- `moveObject()` → 调用 `resolveMovement()`
- `collidesWithSolid()` → 调用 `collidesAt()`
- `getCollisions()` → 调用 `findCollisions()`

这让物理逻辑足够独立，也方便单元测试。

## 7. 场景对象与物理

场景对象不仅是“静态背景”，还可能具备：

- 可破坏
- 可坍塌
- 失去支撑后下落
- 计步后破碎
- 虚空伤害

这些行为依赖碰撞检测，也依赖对象自身的 `update()` 逻辑。

## 8. 相关测试

物理与碰撞相关测试重点验证：

- 玩家不会穿过墙体边缘
- 世界能正确报告被阻挡的位置
- 受击击退不会穿墙
- 启用重力后玩家会下落
- 互补色危险方块会伤害玩家
- 普通场景地面不会错误伤害玩家

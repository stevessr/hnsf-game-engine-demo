# 渲染系统详细分析

## 1. 模块职责

渲染系统负责把游戏世界变成可视图像。

主要组成：

- `SwingGamePanel`
- `Camera`
- `LightingManager`
- `BackgroundAssets`
- `SpriteAssets`
- `TextureManager`
- 覆盖层：`ControlHintsOverlay`、`GoalOverlay`、`MinimapOverlay`、`DebugManager`

## 2. `SwingGamePanel`

这是主窗口渲染面板，也是游戏帧循环的入口。

### 2.1 初始化阶段

构造时会做这些事：

- 创建输入控制器
- 创建设置仓库
- 创建摄像机
- 绑定到 `GameWorld`
- 加载持久化设置
- 注册键盘和鼠标监听器
- 准备背景图与菜单图
- 创建 Swing `Timer`

### 2.2 帧循环

每一帧会执行：

1. 计算帧时间 `deltaSeconds`
2. 处理输入
3. 更新 AI 测试器（如果启用）
4. 更新世界
5. 让摄像机跟随玩家
6. 清空瞬时输入状态
7. 请求重绘

### 2.3 输入与渲染坐标转换

`SwingGamePanel` 的输入监听不是直接使用屏幕坐标，而是先根据缩放比例和视口偏移换算到逻辑坐标。

这样做的好处是：

- 画面可以缩放
- 鼠标点击仍然准确
- 菜单与对话框在屏幕空间下工作正常

### 2.4 画面渲染流程

`paintComponent()` 中，渲染顺序大致为：

1. 背景层
2. 世界层
3. 状态背景蒙版
4. UI 层
5. 游戏 HUD
6. 调试信息
7. 小地图与目标提示

这种顺序清晰地区分了“世界内容”和“界面内容”。

## 3. 视口与摄像机

### 3.1 `Camera`

摄像机保存：

- 视口宽高
- 世界中的跟随位置

`Camera.update()` 会把视角对准目标对象，并尽量保持视口在合理范围内。

### 3.2 摄像机与世界渲染

世界层渲染时会先做平移变换，再绘制对象和光照。

这样玩家看到的就是“摄像机视角”，而不是固定左上角坐标。

## 4. 背景系统

### 4.1 `BackgroundAssets`

它负责：

- 按关卡名解析资源路径
- 加载主菜单和暂停菜单背景
- 加载关卡背景图
- 提供缓存
- 提供 base64 资源数据

### 4.2 背景绘制

`GameWorld` 会缓存背景渲染结果。

背景模式包括：

- 纯色
- 渐变
- 图片

而 `MapBackgroundPreset` 提供具体的主题风格，比如：

- 默认
- 森林
- 洞穴
- 冰原
- 火山
- 沙漠
- 天空
- 夜晚
- 废墟

## 5. 光照系统

### 5.1 `LightingManager`

它支持：

- 环境光
- 光照倍率
- 光源列表
- 探索模式
- 视野迷雾缓存

### 5.2 光照渲染

光照不是简单的“覆盖一层黑色”，而是会根据光源位置和遮挡物绘制光锥/阴影效果。

### 5.3 探索模式

探索模式可以保留已经见过的区域。

这使得场景具有“迷雾探索”的感觉，而不是每帧完全重置。

## 6. UI 覆盖层

### 6.1 提示层

`ControlHintsOverlay` 用于显示控制说明。

### 6.2 目标层

`GoalOverlay` 用于提示当前关卡目标和完成状态。

### 6.3 小地图层

`MinimapOverlay` 用于显示地图缩略图和玩家位置。

### 6.4 调试层

`DebugManager` 可以绘制调试信息，也支持调试控制台。

## 7. 状态背景

当前状态为 `MENU` 或 `PAUSED` 时，面板会叠加菜单背景图。

这让暂停界面和主菜单有明显的视觉区分。

## 8. 相关测试

渲染相关测试主要覆盖：

- 背景缓存是否在预设变化后刷新
- 主菜单和暂停背景是否能渲染
- 场景对象是否会覆盖背景
- 角色与怪物渲染是否符合预期

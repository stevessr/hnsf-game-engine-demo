# 持久化详细分析

## 1. 模块职责

持久化模块负责：

- 地图存储
- 地图加载
- JSON 导入导出
- 设置保存
- 程序化关卡生成

主要类：

- `MapRepository`
- `MapDataMapper`
- `SettingsRepository`
- `ProceduralLevelGenerator`
- `NoiseGenerator`

## 2. `MapRepository`

这是基于 SQLite 的地图仓库。

### 2.1 默认位置

默认数据库文件为：

`~/.hnsfgame/maps.db`

### 2.2 支持能力

- 保存或更新地图
- 按 ID 加载
- 按名称加载
- 列出地图名

### 2.3 数据表

#### `maps`

保存地图级信息：

- 名称
- 尺寸
- 背景颜色
- 背景模式
- 背景预设
- 背景图片名与数据
- 重力开关
- 重力强度

#### `map_objects`

保存对象级信息：

- 类型
- 名称
- 坐标
- 尺寸
- 颜色
- 固体属性
- 背景属性
- 材质与贴图
- `extra_json`

### 2.4 表结构演进

仓库初始化时会检查并补齐缺失列。

这说明项目考虑到了后续功能扩展，而不是只做一次性建表。

## 3. `MapDataMapper`

它是整个数据流的枢纽。

### 3.1 从世界导出

`GameWorld` → `MapData`

导出内容包括：

- 世界尺寸
- 背景
- 重力
- 胜利条件
- 对象列表

### 3.2 导出 JSON

`MapData` → `JSONObject`

适用于：

- 文件保存
- 导入导出
- 外部分享

### 3.3 从 JSON 导入

`JSONObject` → `MapData`

导入时会处理：

- 缺省值
- 背景模式与预设
- 背景图片编码
- 对象数组

### 3.4 应用到世界

`MapData` → `GameWorld`

这个过程会：

- 清空旧对象
- 重建背景
- 恢复重力与胜利条件
- 恢复对象和玩家

## 4. `SettingsRepository`

设置持久化负责把运行参数保存到 JSON 文件中。

保存的内容包括：

- FPS
- 字体大小
- 分辨率
- 油门与减速度
- 重力
- 光照
- 音效
- 按键绑定

这让程序的体验更接近“可配置应用”而不是纯 demo。

## 5. 程序化生成

### 5.1 `ProceduralLevelGenerator`

支持两类程序化关卡：

- 森林
- 洞穴

生成目标不是只放随机块，而是构建可玩结构：

- 安全出生点
- 可落脚地形
- 目标点
- 敌人和物品分布
- 虚空区域等危险地带

### 5.2 `NoiseGenerator`

这是程序化地形的基础噪声实现，提供：

- 一维噪声
- 二维噪声

适合做地形起伏和随机变化。

## 6. 导入导出与编辑器联动

编辑器生成的世界可以直接：

1. 转成 `MapData`
2. 保存进 SQLite
3. 导出成 JSON
4. 再导入回世界

这条闭环是项目里很重要的工程能力体现。

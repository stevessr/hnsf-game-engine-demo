# 编辑器与持久化说明

## 1. 地图编辑器

### `EditorWindow`

`EditorWindow` 是编辑器主窗口，负责把以下内容组织到一个桌面应用中：

- 地图基础属性
- 对象类型选择
- 对象属性编辑
- 关卡预览
- 保存 / 加载 / 导入 / 导出

窗口中可以看到大量控件，例如：

- `JComboBox`
- `JSpinner`
- `JCheckBox`
- `JToggleButton`
- `JTextField`

这说明编辑器已经覆盖了对象和地图的大部分可配置字段。

### `MapEditorController`

这是编辑器的行为核心，支持三种模式：

- `SELECT`：选择对象
- `BUILD`：放置对象
- `ERASE`：删除对象

它还负责：

- 鼠标拖拽
- 对象缩放
- 网格吸附
- 撤销 / 重做
- 选择变化回调
- 保存回调

### 编辑器辅助类

- `EditorGamePanel`：编辑器预览面板
- `EditorOverlay`：网格、选中对象、模式提示等覆盖信息
- `EditorBounds`：编辑边界
- `LevelAutoScanner`：自动扫描可用关卡

## 2. SQLite 持久化

### `MapRepository`

`MapRepository` 负责地图的 SQLite 持久化，默认数据库文件位于：

`~/.hnsfgame/maps.db`

它支持：

- 保存或更新地图
- 按 ID 加载地图
- 按名称加载地图
- 列出全部地图名称

### 数据表结构

数据库里主要有两张表：

#### `maps`

保存地图级别元数据：

- `id`
- `name`
- `width`
- `height`
- `background_color`
- `background_preset`
- `background_mode`
- `background_image_name`
- `background_image_data`
- `gravity_enabled`
- `gravity_strength`

#### `map_objects`

保存地图对象数据：

- `id`
- `map_id`
- `type`
- `name`
- `x`
- `y`
- `width`
- `height`
- `color`
- `solid`
- `background`
- `texture_path`
- `material`
- `extra_json`

`extra_json` 是这个设计里非常重要的字段，它让对象的类型专属属性可以保持扩展性。

## 3. 地图数据映射

### `MapDataMapper`

`MapDataMapper` 在三个世界之间做转换：

1. `GameWorld`
2. `MapData` / `ObjectData`
3. `JSONObject`

它支持：

- 从运行时世界导出为数据对象
- 将数据对象导出为 JSON
- 从 JSON 导入数据对象
- 将数据对象还原为运行时世界
- 将数据对象应用到已有世界

### 图片与背景处理

`MapDataMapper` 还负责：

- 背景图像编码 / 解码
- 背景模式与预设的恢复
- 地图图像名称的保存

## 4. 设置持久化

### `SettingsRepository`

编辑器和游戏运行参数会被单独保存，包括：

- 分辨率
- 帧率
- 字体大小
- 重力
- 光照
- 音量
- 按键绑定

这让游戏的使用体验具有一定的“下次启动继续生效”能力。

## 5. 程序化关卡

### `ProceduralLevelGenerator`

用于生成程序化地图，例如：

- 森林关卡
- 洞穴关卡

### `NoiseGenerator`

通常作为地形生成的噪声基础，用于构建更自然的起伏和分布。

## 6. 导入导出流程

### 导出

`GameWorld` → `MapDataMapper.fromWorld()` → `MapDataMapper.exportToJson()` → 文件保存

### 导入

JSON 文件 → `MapDataMapper.importFromJson()` → `MapRepository.saveMap()` → `MapDataMapper.applyToWorld()`

这条链路保证了地图既能落到 SQLite，也能通过 JSON 共享。

# `MapRepository` 类级分析

## 1. 类职责

`MapRepository` 是基于 SQLite 的地图仓库。

它负责把地图元数据和对象列表持久化到本地数据库，并在需要时重新加载。

## 2. 默认数据库

默认位置：

`~/.hnsfgame/maps.db`

仓库初始化时会自动创建目录和表结构。

## 3. 核心能力

- 保存或更新地图
- 按 ID 加载地图
- 按名称加载地图
- 列出地图名

## 4. 保存流程

### `saveMap(MapData mapData)`

保存时会：

1. 打开 SQLite 连接
2. 开启事务
3. 插入或更新 `maps`
4. 删除旧对象记录
5. 批量插入新对象
6. 提交事务

这保证了地图数据和对象列表的一致性。

## 5. 读取流程

### `loadMapById()` / `loadMapByName()`

读取时会：

1. 查找地图元数据
2. 根据地图 ID 读取对象列表
3. 组合成完整 `MapData`

## 6. 表结构

### 6.1 `maps`

保存地图级属性：

- 名称
- 宽高
- 背景颜色
- 背景模式
- 背景预设
- 背景图片
- 重力开关
- 重力强度

### 6.2 `map_objects`

保存对象级属性：

- 类型
- 名称
- 位置
- 尺寸
- 颜色
- 固体/背景属性
- 材质/贴图
- 扩展 JSON

## 7. 架构特点

### 7.1 事务化

保存过程使用事务，避免地图保存到一半失败导致数据不一致。

### 7.2 兼容升级

初始化时会检查列是否存在，并自动补列，说明它考虑了后续版本演进。

### 7.3 数据与对象分离

数据库只存 DTO，不直接存运行时对象，便于扩展和序列化。

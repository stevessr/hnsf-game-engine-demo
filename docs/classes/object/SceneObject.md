# `SceneObject` 类级分析

## 1. 类职责

`SceneObject` 是场景块的基础实现。

它不只是静态方块，还可以表现为：

- 地面
- 墙
- 背景块
- 虚空
- 云
- 树
- 标牌

## 2. 核心属性

- `solid`
- `background`
- `destructible`
- `durability`
- `collapseWhenUnsupported`
- `collapseDamage`
- `breakAfterSteps`
- `stepCount`
- `collapsing`

## 3. 行为

### 3.1 可破坏

`applyStructuralDamage()` 会减少耐久度，归零时失活。

### 3.2 失去支撑后坍塌

如果启用了坍塌逻辑，场景块在失去支撑后会开始下落并造成伤害。

### 3.3 计步破碎

某些块在被踩踏若干次后会破碎，适合做机关地面。

### 3.4 虚空效果

虚空块会直接对与其重叠的角色/实体造成致命影响。

## 4. 更新逻辑

`update()` 会根据场景类型分别处理：

- 虚空
- 计步破碎
- 坍塌逻辑

这让场景块从“纯绘制对象”变成了“有规则的关卡实体”。

## 5. 渲染逻辑

场景对象渲染时会优先使用素材系统，否则回退到几何绘制。

不同材质或命名会触发不同外观：

- 云朵
- 树
- 标牌
- 虚空

## 6. 设计意义

这个类是地图编辑器和关卡玩法交汇的关键节点。

它决定了“地形不仅能看，还能互动、能破坏、能坍塌”。

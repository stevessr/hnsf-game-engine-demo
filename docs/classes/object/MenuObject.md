# `MenuObject` 类级分析

## 1. 类职责

`MenuObject` 是游戏菜单的统一实现。

它可以用于：

- 主菜单
- 暂停菜单
- 选项菜单
- 结算菜单
- 失败菜单

## 2. 核心状态

- `title`
- `subtitle`
- `options`
- `selectedIndex`
- `hoveredIndex`
- `fontSize`
- `optionColumns`
- `maxVisibleRows`
- `scrollOffset`

## 3. 交互能力

### 3.1 选中与悬停

菜单同时支持键盘选中和鼠标悬停。

### 3.2 翻页与滚动

如果选项很多，可以通过滚动条展示部分内容。

### 3.3 点击命中

`findOptionIndexAt()` 能根据鼠标位置判断点击了哪一项。

## 4. 布局计算

菜单会根据：

- 标题区高度
- 行高
- 列数
- 可见行数
- 滚动偏移

动态计算每个选项的边界和位置。

## 5. 渲染

菜单渲染通常包括：

- 半透明背景板
- 标题
- 副标题
- 选项列表
- 高亮项
- 滚动条

这种布局让菜单既适合键盘导航，也适合鼠标操作。

## 6. 设计价值

菜单对象把“菜单数据”和“菜单行为”统一到了一个可渲染对象里，状态机只需要控制它显示什么，而不用管怎么画。

# 渲染与资源深度分析

这一层关注“图片从哪来”“怎么缓存”“怎么在画面上叠加 UI”。

## 1. 背景资源

### `BackgroundAssets`

- 负责读取关卡背景和菜单背景资源。
- 内部维护“关卡名 -> 资源路径”的映射，例如教程、示例图、森林、废墟、洞穴、Boss 关等。
- 对背景图片使用双缓存：
  - `IMAGE_CACHE`：`BufferedImage`
  - `BASE64_CACHE`：用于 `MapData` 的导出
- 支持两种应用方式：
  - 直接应用到 `GameWorld`
  - 写入 `MapData` 供保存/导出使用
- 对 `procedural-forest`、`procedural-cave` 这类程序化名称做特殊解析。

## 2. Sprite 与贴图资源

### `SpriteAssets`

- 负责角色、怪物、物品、墙体、场景块、Voxel 的精灵绘制。
- 通过 `CACHE` 缓存已加载的资源图片。
- 核心绘制接口包括：
  - `drawPlayer()`
  - `drawMonster()`
  - `drawItem()`
  - `drawWall()`
  - `drawScene()`
  - `drawVoxel()`
- 会根据对象的 `name`、`material`、`monsterKind`、`kind` 做资源推断。
- 如果没有精灵资源，就回退到 `TexturePaint` 或几何绘制。

### `TextureManager`

- 用于加载“自定义路径”的本地图片纹理。
- 以路径字符串作为缓存 key。
- `createTexturePaint()` 会把图片包装成 `TexturePaint`，供方块、地砖、装饰物重复铺贴。

## 3. HUD 与覆盖层

### `ControlHintsOverlay`

- 在画面右下角显示当前状态下可用的快捷键提示。
- 会根据 `GameState` 自动切换提示内容。
- 如果 AI 自动测试开启，会额外显示 `AI ACTIVE`。

### `GoalOverlay`

- 在画面左上角显示当前关卡目标和进度。
- 只有在 `world.isShowGoals()` 为真时才渲染。
- 适合显示“到达终点”“击杀数量”“收集数量”等通关条件。

### `MinimapOverlay`

- 在右上角渲染小地图。
- 通过世界坐标缩放成缩略图。
- 使用不同颜色标记玩家、敌人、目标、物品、墙体和触发器。
- 还会用白框标出摄像机当前可见范围。

## 4. 与上一层的关系

上一层已经单独分析了：

- `Camera`
- `LightingManager`
- `SwingGamePanel`

它们和本层的关系是：

- `Camera` 决定“从世界哪一块开始画”
- `LightingManager` 决定“画面有多亮”
- `SwingGamePanel` 负责把背景、世界层、光照层、HUD 组合起来

## 5. 设计要点

- 资源类统一使用缓存，减少每帧磁盘或类路径 I/O。
- 背景、精灵、HUD 分层清晰，避免渲染逻辑互相污染。
- 大部分绘制逻辑都允许失败回退，保证资源缺失时仍可运行。

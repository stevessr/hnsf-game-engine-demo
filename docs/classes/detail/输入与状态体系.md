# 输入与状态体系深度分析

这一层主要描述“按键 / 鼠标输入如何变成游戏动作”，以及“动作如何被状态机消费”。

## 1. 输入采集层

### `KeyboardManager`

- 维护当前按下的按键集合。
- 维护“本帧刚按下”的一次性集合。
- 提供 `pressKey()`、`releaseKey()`、`isPressed()`、`wasPressedThisFrame()`。
- 每帧结束时通过 `clearTransientStates()` 清理瞬时状态。

### `MouseManager`

- 维护鼠标按键状态与“本帧点击”状态。
- 记录当前鼠标坐标 `mouseX / mouseY`。
- 支持 `pressButton()`、`releaseButton()`、`moveTo()`。
- 与键盘管理器一样，提供 `clearTransientStates()` 和 `reset()`。

## 2. 动作映射层

### `InputAction`

输入语义枚举，抽象成：

- 移动
- 斜向油门 / 机动
- 菜单导航
- 暂停 / 对话推进
- 切换弹种、跳跃、射击、冲刺
- 建造 / 销毁方块
- 切换目标显示

### `InputActionMapper`

- 用 `EnumMap<InputAction, Set<Integer>>` 管理键盘绑定。
- 用 `EnumMap<InputAction, Set<Integer>>` 管理鼠标绑定。
- 默认映射支持：
  - WASD 与方向键
  - Q/E 菜单切换
  - Enter / Space 确认
  - Esc / P 暂停
  - K / 鼠标左键射击
  - Shift 冲刺
  - T 切换油门
  - G 显示目标
  - 鼠标中键建造、右键销毁
- 提供 `isActive()`、`isJustActivated()` 等查询接口，供输入控制器统一消费。

## 3. 运行时协同

### `GameInputController`

这个类已经在上一层单独分析过，这里只强调它的职责分工：

- 先把 `KeyboardManager` / `MouseManager` 的状态翻译成 `InputAction`
- 再分别处理玩家移动、菜单导航、Voxel 建造/销毁
- 最后通过 `finishFrame()` 清除瞬时状态

它是“原始输入 -> 游戏行为”的总入口。

### `KeyBindingsWindow`

- 独立的按键绑定界面。
- 允许用户为每个 `InputAction` 重新绑定键位。
- 改键后会同步到 `SwingGamePanel`，并触发设置保存。
- 说明了输入体系不仅运行时可用，也支持用户配置。

## 4. 状态与运行上下文

### `GameState`

状态枚举把游戏分成：

- `MENU`
- `PLAYING`
- `DIALOG`
- `PAUSED`
- `GAMEOVER`
- `SETTLEMENT`

并提供“是否允许玩家移动 / 世界更新 / 菜单导航 / 对话交互”的判定方法。

### `GameStateMachine`

- 抽象状态切换接口。
- 负责 `transitionTo()`、`canTransitionTo()`、`processInput()`。
- 允许实现类把“状态切换”和“输入消费”分离。

### `GameStateContext`

- 把 `GameWorld`、`GameInputController`、`GameSettings`、`GameRuntimeActions`、`deltaSeconds` 打包传入状态机。
- 避免状态机方法签名无限膨胀。
- 也让测试更容易构造最小上下文。

### `GameRuntimeActions`

- 提供超出状态切换本身的运行时回调。
- 包括：
  - 退出程序
  - 加载关卡
  - 加载下一关
  - 打开编辑器
  - 生成程序化关卡
- `noOp()` 允许在不支持这些动作时安全降级。

### `GameSettings`

- 这是状态机能调整的“运行配置接口”。
- 覆盖 FPS、油门、减速度、重力、分辨率、UI 字体、光照、音效、调试模式等参数。
- 同时提供 `savePersistentSettings()`，让设置更改可持久化。

## 5. 设计要点

- 输入层把“按键码”与“游戏动作”分离，便于换键位。
- 瞬时状态和持续状态分开，避免“按一下”和“按住”混淆。
- 状态上下文统一承载运行所需依赖，减少类之间的硬耦合。
- 这套体系也是 `SwingGamePanel`、`DefaultGameStateMachine` 和 `GameInputController` 协作的基础。

# 编辑器与生成深度分析

这一层把关卡编辑器、地图检查器、关卡管理和程序化生成一起看。

## 1. 编辑器运行时

### `EditorWindow`

- 关卡编辑器主窗口。
- 由三块核心区域组成：
  - 左侧：选择与对象属性编辑
  - 中间：地图预览
  - 右侧：地图设置 / 对象设置 / 工具标签页
- 直接持有：
  - `GameWorld`
  - `EditorOverlay`
  - `EditorGamePanel`
  - `MapEditorController`
  - `MapRepository`
- 负责把 UI 控件值同步到世界对象，并把地图保存到仓库。

### `EditorGamePanel`

- 编辑器的预览画布。
- 会把世界缩放到面板中央，按比例显示。
- 预览时会先调用 `world.render()`，再叠加 `EditorOverlay`。

### `EditorOverlay`

- 编辑器叠加层。
- 负责：
  - 网格显示
  - 选中框高亮
  - 缩放句柄
  - 对象信息提示
  - 快捷键提示板
- 它的职责很纯粹：只画辅助信息，不改世界逻辑。

### `EditorBounds`

- 编辑器的边界修正工具。
- 统一处理：
  - 矩形最小尺寸
  - 位置钳制
  - 物体拖拽时不越界
- `normalizeRect()` 与 `normalizePosition()` 是核心入口。

## 2. 地图扫描与质量检查

### `LevelAutoScanner`

- 用于快速扫描关卡配置问题。
- 会检查：
  - 是否缺少玩家
  - 是否缺少终点
  - 胜利条件是否可达
  - 对象尺寸是否合法
  - 对象是否越界
  - 固体对象是否重叠
  - 名称是否重复
  - 触发器目标是否存在
  - 刷怪笼参数是否合理
- 最后还会输出一条对象统计信息，便于报告使用。

## 3. 关卡管理与切换

### `LevelManager`

- 维护内置关卡列表。
- 支持：
  - 加载指定关卡
  - 加载下一关
  - 重开当前关
  - 生成关卡数据
- 内置模板包括：
  - `tutorial`
  - `demo-map`
  - `level-1` ~ `level-4`
  - `procedural-forest`
  - `procedural-cave`
  - `air-raid-demo`
  - `showcase-demo`
- 它会把构造好的 `GameWorld` 转回 `MapData`，再通过 `MapDataMapper` 应用到运行时世界。

### `WinConditionType`

关卡胜利条件枚举，包含：

- 到达终点
- 消灭所有怪物
- 消灭指定数量怪物
- 收集指定数量物品
- 清除所有物品

它是 `LevelManager`、`GameWorld`、`DefaultGameStateMachine` 和 `LevelAutoScanner` 的共同依据。

## 4. 程序化生成

### `ProceduralLevelGenerator`

- 提供森林和洞穴两种程序化模板。
- `generateForest()`：
  - 使用噪音生成起伏地形
  - 放置草皮、树、怪物、光球
  - 安排玩家与出口
- `generateCave()`：
  - 用噪音填充洞穴墙体
  - 在低密度区域放置怪物
  - 通过支持点搜索找到玩家出生位置
- 内部还包含：
  - 支持生成范围搜索
  - 可通行区域判断
  - 支撑面检测

### `NoiseGenerator`

- 简单的 1D / 2D 平滑噪音实现。
- 采用类似 Perlin Noise 的 `fade / lerp / grad` 结构。
- `ProceduralLevelGenerator` 依赖它生成更自然的地形起伏和洞穴空洞。

## 5. 编码与文本工具

### `Unicode2Gbk`

- 处理 GBK / GB2312 环境下的中文显示问题。
- `convert()` 会在 GBK 环境中尝试把字符串重新编码，减少乱码。
- `isGbkEnvironment()` 用于快速判断当前环境。

## 6. 设计要点

- 编辑器、扫描器、生成器、关卡管理器四者互相配合：
  - 编辑器负责人工编辑
  - 扫描器负责质量检查
  - 生成器负责批量造图
  - 关卡管理器负责加载和切换
- `WinConditionType` 把“关卡目标”统一成枚举，避免散落在各处的魔法字符串。
- `Unicode2Gbk` 是跨编码环境的兼容补丁，适合课程机房场景。

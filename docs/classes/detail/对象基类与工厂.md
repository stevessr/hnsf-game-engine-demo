# 对象基类与工厂深度分析

这一层把对象系统继续拆开，重点看“公共基类”“序列化工厂”“基础可交互对象”。

## 1. 对象接口与公共基类

### `GameObject`

- 所有运行时对象的最小共同接口。
- 统一规定：
  - 名称
  - 类型
  - 坐标与尺寸
  - 颜色
  - 激活状态
  - `update()` / `render()`
- 提供 `updateInactive()` 默认空实现，方便可复活、可计时的对象使用。

### `BaseObject`

- 所有具体对象最常继承的公共基类。
- 持有通用字段：
  - `type`
  - `name`
  - `x / y / width / height`
  - `active`
  - `color`
  - `texturePath`
  - `material`
  - `velocityY`
- 负责名称归一化、尺寸下限、颜色默认值等基础规则。
- 让派生类只关心自己的业务逻辑。

### `ActorObject`

- 角色类对象的通用基类。
- 持有：
  - `health / maxHealth`
  - `attack`
  - `speed`
  - 死亡动画状态
- 提供：
  - `takeDamage()`
  - `heal()`
  - `startDeathAnimation()`
  - `updateDeathAnimation()`
  - `renderInfo()`
  - `revive()`
- 是玩家、怪物这类“可战斗实体”的共同上层抽象。

## 2. 序列化与反序列化工厂

### `GameObjectFactory`

- 负责 `GameObject <-> ObjectData` 的双向转换。
- `toObjectData()` 会把通用字段写入 DTO，并根据具体对象补充 `extraJson`。
- `fromObjectData()` 会根据 `GameObjectType` 创建正确的运行时对象。
- 支持的对象类型包括：
  - `PLAYER`
  - `MONSTER`
  - `ITEM`
  - `TRIGGER`
  - `SPAWNER`
  - `VOXEL`
  - `GOAL`
  - `PROJECTILE`
  - `WALL`
  - `BOUNDARY`
  - `MENU`
  - `DIALOG`
  - `SCENE`
- `extraJson` 用来存对象专属字段，例如：
  - 玩家等级、经验、弹种
  - 怪物种类、远程/飞行/复活参数
  - 物品种类、数值、消息
  - 触发器动作与目标
  - 刷怪笼参数
  - 菜单选项
  - 对话文本
  - 投射物速度和伤害
  - 场景块的可破坏信息

### `ColorUtils`

- 提供颜色互补判断。
- `complement()` 可快速求补色。
- `isComplementary()` 使用 HSB 色相差判断是否为互补色。
- 该工具类被玩家与危险方块的颜色冲突逻辑使用。

## 3. 基础世界对象

### `BoundaryObject`

- 关卡边界对象，用于封住地图四边。
- 提供 `top / bottom / left / right` 工厂方法。
- 渲染上带有警示色和斜线纹理，便于识别地图边界。

### `GoalObject`

- 关卡终点 / 出口对象。
- 带有脉冲光晕和粒子环绕动画。
- 通过 `update()` 只推进动画计时器，核心逻辑保持简单。

### `ItemObject`

- 可拾取物品。
- 支持多个种类：
  - 回复类
  - 速度类
  - 光照类
  - 经验类
  - 护盾类
- 失活后可根据 `kind` 自动恢复，光源类物品默认可周期复活。

### `SpawnerObject`

- 刷怪笼对象。
- 通过时间累积按波次生成怪物。
- 具有：
  - `monsterKind`
  - `spawnIntervalSeconds`
  - `maxAlive`
  - `spawnWaveSize`
  - `spawnRadius`
  - `spawnOffsetX / spawnOffsetY`
  - `spawnGroupId`
- 生成时会给怪物打上分组标记，便于统计同一刷怪源。

### `TriggerObject`

- 触发器对象。
- 玩家进入范围后，根据 `TriggerAction` 对目标对象或世界做操作。
- 支持：
  - 激活 / 失活 / 切换对象
  - 设置重生点
  - 显示 / 隐藏 / 切换目标显示
  - 让玩家立即复活
- 支持一次性触发和重复触发。

### `VoxelObject`

- 体素 / 地砖类对象，通常用于可拼装关卡。
- 优先使用纹理或精灵资源渲染。
- 如果没有资源，则按材料色进行几何绘制。

### `WallObject`

- 墙体对象，通常作为固体障碍。
- 优先使用 `SpriteAssets` 渲染。
- 如果是可破坏墙，会额外绘制交叉标记。

## 5. 设计要点

- `GameObject` 只定义最少接口，便于不同对象统一管理。
- `BaseObject` 把“位置 / 尺寸 / 名称 / 颜色 / 材质”这些公共字段收拢起来。
- `GameObjectFactory` 让持久化层、编辑器、运行时之间共享同一套对象描述。
- `TriggerObject`、`SpawnerObject`、`ItemObject` 让关卡设计具有更高的脚本化能力。

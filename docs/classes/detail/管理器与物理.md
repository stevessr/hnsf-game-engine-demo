# 管理器与物理深度分析

这一层关注运行时的“对象调度”“音效”“调试”“自动化 AI”和“碰撞 / 移动解析”。

## 1. 对象调度与生命周期

### `EntityManager`

- 保存世界中的全部对象。
- 用 `EnumMap<GameObjectType, List<GameObject>>` 做类型缓存。
- 使用 `cacheDirty` 脏标记避免每帧重复分类。
- 提供：
  - `add()`
  - `remove()`
  - `clear()`
  - `getObjects()`
  - `getActiveObjects()`
  - `getObjectsByType()`
  - `findFirstByType()`
  - `getSolidScenes()`
- `updateAll()` 会复制对象列表后再遍历，避免在更新中增删对象造成并发修改问题。
- `renderWorld()` 和 `renderUI()` 按层级调用对象渲染。

## 2. 音效与调试

### `SoundManager`

- 简易 WAV 音效管理器。
- 通过“音效名 + 别名 + 变体”机制寻找资源。
- 内部分组包括：
  - 受伤类
  - 射击类
  - 菜单类
  - 其他效果类
- 支持主音量和分类音量独立控制。
- 使用 `Clip` 播放，播放结束后自动关闭资源。

### `DebugManager`

- 负责屏幕上的调试信息显示。
- 显示内容包括：
  - FPS
  - 玩家坐标与速度
  - 玩家血量
  - 世界对象总数
  - 最近日志
- 支持内置指令：
  - `god`
  - `speed`
  - `heal`
  - `light`
  - `tp`
- 很适合课程展示时演示“开发者后门”和“实时调试能力”。

### `AITestManager`

- 自动测试 AI。
- 会根据当前胜利条件选择目标：
  - 收集物品
  - 击杀怪物
  - 追向终点
- 具有基础避障、自动跳跃和自动射击逻辑。
- 适合做 UI/关卡回归测试，也适合作为“演示模式”。

## 3. 物理与碰撞

### `Aabb`

- 轴对齐边界框。
- 用于快速碰撞测试。
- 支持：
  - 从对象创建
  - 指定坐标创建
  - `intersects()` 检测相交

### `MovementResult`

- 表示一次移动解析的结果。
- 记录：
  - 最终坐标
  - X / Y 是否被阻挡
  - 分别被哪个对象阻挡
- `isBlocked()` 可以快速判断是否任一轴受阻。

### `PhysicsEngine`

- 负责移动解析和碰撞判定。
- `resolveMovement()` 会：
  - 先把目标坐标限制到世界边界内
  - 计算移动范围盒，缩小障碍物候选集合
  - 分别沿 X / Y 轴逐步推进，防止穿墙
- 还提供：
  - `collidesAt()`
  - `findCollisions()`
- 这是 `GameWorld` 和对象移动时最核心的底层工具。

## 4. 辅助窗口

### `KeyBindingsWindow`

- 也属于输入子系统的 GUI 工具。
- 它在这里单独提一下，因为它会直接修改 `InputActionMapper`。
- 主要职责：
  - 展示每个动作的当前绑定
  - 捕获新的按键
  - 调用 `SwingGamePanel.savePersistentSettings()`

## 5. 设计要点

- `EntityManager` 把对象增删、类型索引和层级渲染统一起来。
- `SoundManager` 将声音控制和游戏逻辑解耦。
- `PhysicsEngine` 采用“范围盒 + 逐步推进”组合，兼顾性能与稳定性。
- `DebugManager` / `AITestManager` 都属于“开发辅助层”，但在教学演示中很有价值。

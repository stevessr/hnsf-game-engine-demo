# `DefaultGameStateMachine` 类级分析

## 1. 类职责

`DefaultGameStateMachine` 是游戏流程控制核心。

它把菜单、游戏、对话、暂停、失败和结算统一纳入一套状态转换逻辑。

## 2. 状态管理

### 2.1 当前状态

- `currentState`
- `previousState`

### 2.2 允许转换

使用 `allowedTransitions` 显式定义状态图，避免非法跳转。

### 2.3 对话隐藏

为了让暂停菜单不打断剧情显示，它还维护：

- `hiddenDialog`
- `dismissedDialogNames`

## 3. 核心入口

### `processInput(GameStateContext context)`

这是状态机的主入口。

它会按当前状态分派到不同处理函数：

- 菜单输入
- 游戏输入
- 对话输入
- 暂停输入
- 失败输入
- 结算输入

## 4. 菜单系统

状态机内部直接支持多类菜单：

- 主菜单
- 关卡选择
- 暂停菜单
- 选项菜单
- 失败菜单
- 胜利菜单

并且菜单项不仅仅是“按钮”，还可能触发：

- 切换 FPS
- 切换分辨率
- 切换音量
- 切换光照
- 切换重力
- 进入编辑器
- 生成程序化关卡

## 5. 暂停逻辑

暂停的处理重点是：

1. 隐藏当前对话框
2. 创建暂停菜单
3. 清空玩家移动输入
4. 恢复时还原对话框和状态

这避免了暂停后 UI 和剧情状态混乱。

## 6. 失败与胜利

### 6.1 `GAMEOVER`

失败菜单会展示失败原因，并允许：

- 重生
- 返回菜单

### 6.2 `SETTLEMENT`

胜利菜单会根据是否存在下一关决定：

- 下一关
- 返回菜单

## 7. 运行时动作

状态机不是直接打开窗口或退出程序，而是通过 `GameRuntimeActions` 发出请求。

这是一种典型的“状态机只做决策，外部做执行”的解耦方式。

## 8. 设置调整

`DefaultGameStateMachine` 负责动态修改 `GameSettings` 和 `GameWorld` 的运行参数，包括：

- FPS
- 分辨率
- 油门
- 减速度
- 光照
- 音量
- UI 字体大小

这让菜单具备真正的运行时配置能力。

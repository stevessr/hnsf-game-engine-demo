# `SwingGamePanel` 类级分析

## 1. 类职责

`SwingGamePanel` 是主游戏面板，也是游戏帧循环入口。

它同时承担：

- 输入监听
- 时间步进
- 游戏更新
- 画面渲染
- 设置持久化
- HUD 和调试界面绘制

## 2. 关键依赖

- `GameWorld`
- `GameInputController`
- `SettingsRepository`
- `Camera`
- `ControlHintsOverlay`
- `MinimapOverlay`
- `GoalOverlay`
- `DebugManager`
- `AITestManager`

## 3. 初始化

构造时会：

1. 绑定 `GameWorld`
2. 创建输入控制器
3. 创建摄像机
4. 加载持久化设置
5. 注册键盘鼠标监听
6. 准备菜单背景图
7. 设置 Swing `Timer`

这说明它不是纯视图，而是“视图 + 运行调度器”的组合体。

## 4. 主循环

### `onFrame()`

每一帧的顺序是：

1. 计算 `deltaSeconds`
2. 限制过长帧，避免物理突变
3. 处理输入
4. 更新 AI 测试器
5. 更新世界
6. 摄像机跟随玩家
7. 清除瞬时输入
8. 重绘

这就是整个游戏运行的核心节拍器。

## 5. 输入适配

### 键盘

通过 Swing 的 `InputMap` / `ActionMap` 同步按键绑定，确保“按下/释放”都能被记录。

### 鼠标

鼠标输入会被转换成逻辑坐标，而不是直接用屏幕坐标。

### 空间模式

当处于菜单或对话状态时，鼠标使用屏幕空间；当处于游戏状态时，使用世界空间。

这点对交互精度很关键。

## 6. 渲染结构

渲染顺序大致如下：

1. 世界背景
2. 世界实体
3. 状态背景遮罩
4. UI 覆盖层
5. 控制提示
6. 调试信息
7. 小地图和目标提示

## 7. 设置与持久化

`SwingGamePanel` 还是 `GameSettings` 的实现者。

它负责保存和恢复：

- FPS
- 字体大小
- 分辨率
- 重力开关
- 光照开关
- 音量与分类音量
- 按键绑定

因此用户在设置面板里改动的内容会真正落到持久化文件里。

## 8. 调试与辅助

它还包含：

- F3 调试切换
- H 切换 AI 自动游玩
- 反引号调试控制台

这说明开发者在 UI 层就内建了测试和调试通道。

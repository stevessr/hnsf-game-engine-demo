# `Camera` 类级分析

## 1. 类职责

`Camera` 负责把“大世界”裁剪成“当前视口”。

它不直接管理对象，而是管理：

- 视口宽高
- 摄像机在世界中的位置
- 对目标对象的跟随

## 2. 核心状态

- `x`
- `y`
- `viewportWidth`
- `viewportHeight`

其中 `x`、`y` 使用浮点保存，便于平滑跟随。

## 3. 主要行为

### `update(GameWorld world, GameObject target)`

摄像机会根据目标对象的位置更新自身坐标，并尽量保持在世界边界内。

### `setViewportSize()`

当窗口尺寸变化时，摄像机也需要同步更新视口大小。

## 4. 使用方式

在主渲染面板中，摄像机一般会跟随玩家：

1. 获取玩家对象
2. 调用 `camera.update(world, player)`
3. 渲染时对世界层执行平移

## 5. 设计意义

摄像机把“屏幕看到什么”和“世界实际坐标是什么”分开了。

这对于大地图、滚动视角和 UI 叠加都很重要。

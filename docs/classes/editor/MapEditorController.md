# `MapEditorController` 类级分析

## 1. 类职责

`MapEditorController` 是编辑器交互的核心控制器。

它把鼠标、键盘、对象选择、拖拽、缩放、撤销重做等行为统一起来。

## 2. 编辑模式

### `SELECT`

选择与移动对象。

### `BUILD`

在指定位置创建对象。

### `ERASE`

删除对象。

## 3. 控制器状态

主要状态包括：

- 当前选中对象
- 当前选中类型
- 网格吸附
- 网格大小
- 默认字体大小
- 画笔颜色
- 拖拽偏移
- 缩放手柄
- 上下文命令栈

## 4. 事件绑定

`bind()` 会把以下事件挂到编辑器画布：

- 鼠标按下
- 鼠标释放
- 鼠标拖拽
- 鼠标移动
- 键盘按下

## 5. 选择与编辑

### 5.1 选中对象

`setSelectedObject()` 会同步：

- 当前对象
- 覆盖层显示
- 选择监听器

### 5.2 添加对象

`addObjectOnce()` 会按当前类型和插入点创建一个对象并立即选中。

### 5.3 删除、复制、移动、缩放

控制器都提供了独立方法处理。

这让 UI 层不用直接操作对象集合。

## 6. 撤销 / 重做

编辑器使用命令栈管理历史：

- 添加
- 删除
- 移动
- 缩放
- 属性修改

这是典型的编辑器命令模式。

## 7. 网格与吸附

控制器支持：

- 网格显示
- 网格吸附
- 网格尺寸调整

这使对象布局更规整，也方便课程演示。

## 8. 属性编辑

编辑器不仅支持通用属性，还支持大量对象专属字段，例如：

- 玩家弹种与属性
- 怪物种类与移动参数
- 物品数值
- 触发器动作
- 生成器参数
- 场景块耐久度与倒塌属性

## 9. 设计价值

它让地图编辑从“放置方块”提升为“编辑整个关卡状态机和对象行为”。

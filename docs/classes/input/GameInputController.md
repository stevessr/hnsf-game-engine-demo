# `GameInputController` 类级分析

## 1. 类职责

`GameInputController` 是输入层的“行为中枢”。

它把键盘和鼠标状态转换成：

- 玩家移动
- 射击
- 炸弹蓄力
- 菜单选择
- 体素建造 / 销毁

## 2. 三个基础依赖

- `KeyboardManager`
- `MouseManager`
- `InputActionMapper`

这三者分别负责：

1. 记录按键状态
2. 记录鼠标状态
3. 把物理输入映射成语义动作

## 3. 输入处理入口

### `processInputs(GameStateContext context)`

处理策略分两种：

- 如果世界挂了状态机：交给状态机处理
- 如果没有状态机：直接处理玩家、菜单和体素系统

这让输入系统既适合正式游戏，也适合测试或简化环境。

## 4. 玩家运动

### `applyPlayerMovement()`

该方法负责：

- 切换颜色
- 切换弹种
- 跳跃
- 射击
- 炸弹蓄力
- 切换油门
- 计算方向向量
- 冲刺消耗体力
- 体力恢复

它不是简单的“按键移动”，而是把动作与角色状态结合起来。

## 5. 射击模式

### 5.1 普通射击

按下即发射。

### 5.2 炸弹蓄力

如果当前弹种是 `BOMB`：

- 按住时蓄力
- 松开时发射
- 蓄力时会记录朝向

### 5.3 鼠标瞄准

支持直接朝鼠标位置射击。

## 6. 菜单导航

`applyMenuNavigation()` 负责：

- 上一项 / 下一项
- 鼠标悬停
- 鼠标确认
- 键盘确认

这与状态机的菜单逻辑配合使用。

## 7. 体素系统

`applyVoxelSystem()` 提供一种即时编辑能力：

- 鼠标中键：建造体素
- 鼠标右键：删除体素

它在正式编辑器之外，也给游戏态保留了简易建造能力。

## 8. 每帧收尾

`finishFrame()` 会清空 transient 状态，避免同一按键在下一帧重复触发“一次性动作”。

## 9. 设计意义

这个类的价值在于：它把低层输入和上层玩法动作之间的映射封装好了，使得状态机和游戏对象不必直接处理原始键盘事件。

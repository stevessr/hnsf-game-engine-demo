# 枚举与 DTO 详细说明

## 1. 枚举类

### 1.1 `GameState`

定义游戏运行状态：

- `MENU`
- `PLAYING`
- `DIALOG`
- `PAUSED`
- `GAMEOVER`
- `SETTLEMENT`

并提供状态能力判断：

- 是否允许玩家移动
- 是否允许世界更新
- 是否允许菜单导航
- 是否允许对话交互

### 1.2 `InputAction`

输入语义动作枚举，覆盖：

- 移动
- 方向辅助
- 菜单操作
- 对话推进
- 武器切换
- 体素系统
- 冲刺与油门
- 目标显示

### 1.3 `GameObjectType`

对象大类枚举：

- `PLAYER`
- `MONSTER`
- `SCENE`
- `WALL`
- `BOUNDARY`
- `MENU`
- `DIALOG`
- `ITEM`
- `TRIGGER`
- `SPAWNER`
- `VOXEL`
- `GOAL`
- `PROJECTILE`

### 1.4 `ProjectileType`

投射物类型包含丰富的行为参数，例如：

- 速度倍率
- 伤害倍率
- 是否受重力
- 是否爆炸
- 爆炸半径
- 引信时间
- 光照强度
- 穿透数
- 追踪强度

这使弹种不只是“外观不同”，而是“行为参数不同”。

### 1.5 `MonsterKind`

怪物类型定义了：

- 默认颜色
- 是否飞行
- 默认重力百分比
- 速度倍率

还支持根据名称或材质做推断，方便从地图数据恢复怪物种类。

### 1.6 `TriggerAction`

触发器动作包括：

- 激活
- 关闭
- 切换
- 设置复活点
- 显示/隐藏/切换关卡目标
- 重生玩家

### 1.7 `MapBackgroundMode`

背景模式：

- `GRADIENT`
- `SOLID`
- `IMAGE`

### 1.8 `MapBackgroundPreset`

背景预设：

- 默认
- 森林
- 洞穴
- 冰原
- 火山
- 沙漠
- 天空
- 夜晚
- 废墟

它不仅提供枚举，还提供主题绘制能力。

### 1.9 `WinConditionType`

关卡胜利条件：

- 到达目标
- 击杀所有怪物
- 击杀指定数量怪物
- 收集指定数量物品
- 清空所有物品

## 2. DTO 类

### 2.1 `MapData`

地图级 DTO，保存：

- ID
- 名称
- 尺寸
- 背景
- 重力
- 胜利条件
- 目标计数
- 对象列表

### 2.2 `ObjectData`

对象级 DTO，保存：

- 类型
- 名称
- 坐标
- 尺寸
- 颜色
- 固体属性
- 背景属性
- 材质
- 贴图
- 扩展 JSON

## 3. 为什么 DTO 很重要

DTO 让“运行时对象”与“持久化对象”分离，带来三个好处：

1. 结构清晰
2. 便于存储和导入导出
3. 便于扩展额外字段

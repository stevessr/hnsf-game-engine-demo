# 测试索引

本目录的测试覆盖了项目的主要模块和关键行为。

## 1. 入口与基础

- `org.example.AppTest`
  - 验证入口类的基础辅助方法

## 2. 编辑器

- `lib.editor.EditorBoundsTest`
  - 验证编辑边界归一化与裁剪
- `lib.editor.LevelAutoScannerTest`
  - 验证关卡检查器的告警与通过条件
- `lib.editor.MapEditorControllerTest`
  - 验证编辑器模式、单次创建和交互流程

## 3. 游戏世界与关卡

- `lib.game.GameWorldPhysicsTest`
  - 验证墙体边缘、阻挡位置、重力和碰撞伤害
- `lib.game.LevelManagerAndItemTest`
  - 验证内置关卡、程序化关卡、物品拾取和地图映射

## 4. 输入系统

- `lib.input.GameInputControllerTest`
  - 验证移动、冲刺、菜单、射击、炸弹和体素交互
- `lib.input.InputActionMapperTest`
  - 验证默认映射与解绑行为
- `lib.input.RobotInputIntegrationTest`
  - 验证真实键盘输入能驱动角色移动

## 5. 管理器

- `lib.manager.EntityManagerTest`
  - 验证实体筛选、删除与分类
- `lib.manager.SoundManagerTest`
  - 验证音量控制、静音与资源可用性

## 6. 对象模型

- `lib.object.ColorUtilsTest`
  - 验证互补色和颜色判断
- `lib.object.GameObjectFactoryTest`
  - 验证对象序列化 / 反序列化
- `lib.object.GameObjectModelTest`
  - 验证玩家、怪物、场景、投射物、物品和渲染行为

## 7. 持久化

- `lib.persistence.MapDataMapperTest`
  - 验证世界、DTO、JSON 的完整转换链路
- `lib.persistence.MapRepositoryTest`
  - 验证 SQLite 地图保存与读取
- `lib.persistence.ProceduralLevelGeneratorTest`
  - 验证森林和洞穴程序化生成
- `lib.persistence.SettingsRepositoryTest`
  - 验证设置保存与恢复

## 8. 状态机

- `lib.state.DefaultGameStateMachineTest`
  - 验证菜单、暂停、结算、重启、关卡切换和 UI 居中
- `lib.state.GameStateMachineTest`
  - 验证状态机接口与基本状态转换

## 9. 通用工具

- `lib.utils.Unicode2GbkTest`
  - 验证编码转换相关辅助逻辑

## 10. 测试整体价值

这些测试的覆盖方式说明项目当前已经不只是“能跑”，而是已经对以下关键行为形成了回归保护：

- 输入映射
- 游戏主循环
- 物理与碰撞
- 关卡生成
- 编辑器行为
- 地图持久化
- 状态机
- 对象序列化
